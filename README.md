# Work Log Reviewer

Gemini Nano（オンデバイスLLM）を使って作業ログを自動レビューするAndroidアプリです。
Obsidianのvaultと連携し、日次・週次のレビューをPixel端末上で完結させます。

## 特徴

- **完全オフライン動作** - Gemini NanoをAICore経由でオンデバイス実行。作業ログが外部に送信されません
- **Obsidian連携** - 既存のvaultフォルダを指定するだけで動作
- **外部指示ファイル** - REVIEW.md / PROMPT.md / WEEKLY_PROMPT.md / WIKI.mdでLLMへの指示をカスタマイズ可能
- **長期メモリ** - レビューから得た洞察をmemory.mdに蓄積可能
- **Wiki生成** - 作業ログから内容を抽出してObsidianのWikiページの生成補助が可能

## 動作環境

- Pixel 10Pro（AICore搭載端末）
- Android 16
- Gemini Nanoのダウンロードが完了していること（AICore経由で自動DL）

## 機能一覧

| タブ | 内容 |
|---|---|
| レビュー | 日次作業ログの総評・改善提案・次のアクションを生成 |
| 週次 | 週単位の分析・Q&A形式の振り返りレポートを生成 |
| メモ | 任意のmdファイルをレビュー |
| Wiki | 概念レジストリとWikiページの管理 |
| チャット | REVIEW.mdを参照したフリーチャット |
| 設定 | パス・モデル設定・スケジュール等 |

## セットアップ

1. APKをインストール（リリースページから）
2. 「すべてのファイルへのアクセス」を許可
3. 設定タブで各ファイルのパスを指定
   - 作業ログフォルダ
   - REVIEW.md
   - memory.md
   - Wikiフォルダ

## 外部指示ファイルによるカスタマイズ

アプリの動作はObsidianで編集できる指示ファイルで細かく調整できます。
ファイルが存在しない場合はアプリ内のデフォルト設定で動作します。

| ファイル | 役割 | 調整できる内容 |
|---|---|---|
| REVIEW.md | 評価基準 | レビューの判断軸・評価の視点・具体例 |
| PROMPT.md | 日次・メモの出力指示 | 総評・改善提案・次のアクションの生成方法 |
| WEEKLY_PROMPT.md | 週次の出力指示 | 成果・課題・質問・総合所見の生成方法 |
| WIKI.md | 概念抽出の基準 | 抽出する概念の粒度・優先度・除外ルール |

リビルド不要でObsidianから直接編集できるため、
LLMの出力が気に入らない場合はファイルを書き直すだけで調整できます。

## Gemini Nanoについて

本アプリはGemini Nano（オンデバイス小規模LLM）を使用しています。
クラウドLLMと比べて以下の制約があります。

- **出力トークン数の上限が256** - 長い回答は生成できません
- **指示への追従精度が低い場合がある** - プロンプトを工夫しても意図通りに出力されないことがあります
- **抽象度の高い指示が苦手** - 具体的な例示（✅/❌形式）を指示ファイルに書くと精度が上がります
- **指示文のオウム返しが起きやすい** - REVIEW.mdは宣言文より問いかけ形式で書くことを推奨します

これらの制約を踏まえた上で外部指示ファイルを調整することで、出力品質を改善できます。

## 開発について
